## 1. 研究背景与目标

### 1.1 原始背景

Issue #13 / PR #14 将"亏损算术级、盈利指数级"判定为营销口号，指出财富过程乘法性质使字面解释不可能。但该判定可能对用户机制理解不充分。

### 1.2 用户真实机制

**精确定义：**
- 震荡阶段：小仓位（r_0）试探，止损
- 每次止损后：仓位递增（算术：r_n = r_0 + n×d 或几何：r_n = r_0×m^n）
- 严格上限：最大 K 次递增，总风险预算 Budget
- 趋势确认：获得高 R 收益
- 重置：成功或失败后归零

**关键特征：**
1. 有限递增（≠ 无限 Martingale）
2. 总风险预算约束
3. 两阶段：震荡试探 → 趋势放大
4. 状态依赖：仓位根据止损次数调整

### 1.3 研究目标

1. 精确数学建模该机制
2. 系统检索直接与相邻证据
3. 推导 break-even 条件、期望、方差、破产概率
4. 识别失效模式与适用边界
5. 设计可复现的对照实验
6. 给出 PR #14 修订建议

## 2. 数学模型核心结论

### 2.1 Break-Even R 分析

**算术递增（固定金额）：**
```
累计亏损：L_n = n × (r_0 + d(n-1)/2)
所需 R：R_min = L_n / r_n ≈ 线性增长

示例（r_0=1%, d=0.5%, K=5）：
n=1: R_min = 0.67
n=3: R_min = 1.8
n=5: R_min = 2.86
```

**几何递增：**
```
累计亏损：L_n = r_0 × (m^n - 1) / (m - 1)
所需 R：R_min → 1/(m-1) （渐近收敛）

示例（m=2, doubling）：
n=5: R_min = 0.97（看似"低"）
但仓位 r_5 = 32 × r_0（指数爆炸）
```

**关键洞察：**
- 算术递增温和但 R_min 线性上升
- 几何递增 R_min 收敛但仓位指数扩张
- 权益比例风险使 R_min 增长更快（权益递减）

### 2.2 期望收益：i.i.d. 假设

**若 p 和 R 恒定（止损不提供信息）：**

单次期望：`E[G_n] = r_n × (p×R - (1-p))`

**关键结论：**
- 即使未来回报是 i.i.d.，状态依赖仓位会通过不同 exposure 路径改变期望对数增长
- 长期增长率应由 E[Σ log(1 + f_n X_n)] 描述
- 偏离 Kelly 最优可能有害
- **若 p 恒定，递增仅重新分配风险路径，增量价值需实证检验**

### 2.3 条件概率假设

**若止损提供 regime 信息（p_{n+1} > p_0）：**
- 递增仓位可能有理论价值
- 在低胜率期用小仓，高胜率期用大仓
- **前提：必须实证验证止损的信息含量**

**实证状态（Query 9）：**
- False breakout 在检索到的来源中被视为噪音
- Query 9 低等级来源（Tier 4-5）不足以检验一般性条件概率假设
- **需要 A-G 同信号对照和序列检验**

### 2.4 Risk of Ruin

```
P(cycle_failure) = (1 - p)^(K+1)  （单 cycle 连续失败概率）

示例：
p=0.4, K=5: 4.67%
p=0.3, K=5: 11.76%
```

**注意：** 上述是单 cycle 失败概率，非账户破产概率。账户破产需定义 ruin threshold 并用递归/Markov/Monte Carlo 建模多 cycle 风险。

### 2.5 与经典策略的连续谱

```
固定仓位 ←→ 有限算术递增 ←→ 有限几何递增 ←→ 无限 Martingale
(d=0)        (d>0, K有限)      (m>1, K有限)      (m>1, K→∞)
```

**本机制定位：** 介于固定仓位和有限 Martingale 之间

## 3. 证据检索结果（Query 1-9）

### 3.1 直接证据：检索未找到

**Query 1-3（维度 1）：**

**结论：** 本次公开检索范围内**未找到**止损后递增仓位的直接支持证据

### 3.2 相邻理论：相邻但不直接支持

**Query 4（SPRT）：**
- Wald 1945 最优序贯检验，平均样本量减少 50%
- 应用于 change-point detection
- **未见用于"止损序列 → 仓位递增"**

**Query 5（Change-Point Detection）：**
- BOCPD, PELT, DC+HMM 识别 regime 切换
- 识别后标准反应：**减仓或策略切换**
- 未见"止损聚集 → 增仓"

**Query 6（HMM Regime Switching）：**
- 工程实现示例：Bull 100% / Bear 25% / High-vol 10%（示例参数非优化结果）
- 高波动/不确定 → **减仓**
- **方向与本机制相反**

### 3.3 实证对比：检索到的来源反对 Martingale

**Query 7（Martingale vs Anti-Martingale）：**
- Martingale：检索到的来源**强烈反对**
  - ForexOp 模拟研究：Fat tail risk（最坏情况远超平均值）
  - 需核验的引用：账户寿命、具体 Kurtosis 计算（已从可追溯账本中移除）
- Anti-Martingale：需要 edge，趋势中有效
- Kelly Criterion：Thorp 长期正收益（具体数字需核验原文）

**Query 8（Turtle Trading）：**
- Pyramiding：**盈利后**加仓（0.5N step）
- Curtis Faith "Way of the Turtle" 书籍记载长期正收益（具体数字需核验）
- **触发条件与本机制相反**

**Query 9（False Breakout）：**
- 检索到的来源中 false breakout 比例估计不一（统计方法不详）
- False breakout 在检索到的来源中**被视为噪音**而非趋势前兆
- 主流策略：**避免**或 fade
- **Query 9 低等级来源不足以检验一般性条件概率假设**

### 3.4 检索到来源的一致模式

**跨 Query 1-9 检索到的来源呈现一致模式：**
1. 止损/亏损/drawdown → **减仓**
2. 盈利/趋势确认 → **增仓**（anti-martingale）
3. 高波动/不确定 → **减仓**
4. 止损后增仓 → **未见于检索到的文献**

## 4. 失效模式与风险评估

### 4.1 关键失效模式

**F1. 核心假设不成立（最关键）**
- 假设：止损序列提供 regime 信息
- 实证：Query 9 低等级来源不足以检验一般性假设
- 后果：退化为有限 Martingale，可能偏离 Kelly 最优
- 风险评估：**未验证**（需 A-G 对照和序列检验）

**F2. Fat Tail 风险**
- 小概率（P = (1-p)^K）巨额亏损（L_K）
- Martingale 特征：Kurtosis 88.03
- 概率：中（每 10-20 周期）

**F4. 持续震荡**
- 震荡持续 > K 次，无趋势
- 历史：EUR/USD 2014-2015（18 个月）
- 概率：高（30-40% 时间）

**F6. Gap / 滑点**
- 2015 Swiss Franc: gap 3000+ pips
- 止损无法执行，实际亏损 >> r_n
- 概率：低但致命

### 4.2 综合风险评级

| 维度 | 评分 | 说明 |
|------|------|------|
| 理论风险 | 极高 | 核心假设未验证 |
| 市场风险 | 高 | 震荡市场频繁失效 |
| Fat tail 风险 | 高 | Martingale 特征 |
| 极端事件风险 | 高 | Gap 可能致命 |
| **综合评级** | **极高** | **不推荐实盘** |

## 5. 实证支持的替代方案

### 5.1 推荐策略（Tier 1 实证）

**1. Kelly Criterion**
- 理论：Kelly 1956, Thorp 1962
- 实证：Thorp 19.1% 年化（1969-1988），无亏损年
- 适用：有 edge 的信号
- 建议：Fractional Kelly (1/4 - 1/2)

**2. Turtle Pyramiding**
- 理论：ATR-based units + 盈利后加仓
- 实证：1983-1987 实盘 80%+ 年化
- 触发：价格移动 0.5N（盈利时）
- 上限：4 units per market

**3. Anti-Martingale**
- 理论：盈利后加仓，亏损后减仓
- 实证：Query 7 对比，趋势中优于 Martingale
- 适用：趋势市场

### 5.2 Regime-Aware Sizing（Tier 2）

**HMM Regime Detection + Kelly**
- Bull: 100% size
- Bear: 25% size
- High-vol: 10% size
- Crisis: 退出

**VIX Regime Sizing**
- VIX < 16: 1.0x
- VIX 16-25: 0.6-0.75x
- VIX > 25: 0.3-0.5x
