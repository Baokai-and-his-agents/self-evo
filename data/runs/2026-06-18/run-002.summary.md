## Decisions

- **Single Python implementation + PowerShell wrappers** rather than two parallel
  codebases. The Windows `.ps1` entrypoints resolve `python`/`py -3` and forward
  stdin/args; the three surfaces (hooks, validator, tests) share `_policy.py`
  and `policy.json`, so they cannot diverge. Python 3 is a documented
  prerequisite (the agent does not install global software; no `pip` deps —
  stdlib only, since `pyyaml` is unavailable and must not be globally installed).
- **Mode config** lives in `scripts/hooks/config.json` (protected hook
  implementation), overridable by env `SELF_EVO_ROLLOUT_MODE`. This keeps the
  committed default auditable and prevents a confused worker from silently
  flipping enforcement.
- **`.github/**` and `docs/**` are WARN (proposal_required), not BLOCK**, so
  template edits are visible without trapping the worker.
- **`scripts/**` is protected hook implementation**: in enforce modes a worker
  cannot edit hooks/config via editor tools. Development happens in audit mode
  on an agent branch (the hook inspects tool calls, not commits).

## Known limitations

- Command classification is substring-based → obfuscated/aliased dangerous
  commands may evade detection; benign strings containing the substrings may
  false-positive.
- Secret detection is path/name heuristic → enforce modes can deny reads of
  sensibly-named non-secret files.
- `.ps1` wrappers require Python 3 on PATH; without it they error.
- The validator is deterministic *given live repo state*, not state-independent.
- Root-level infrastructure files (e.g. a root `.gitignore`) fall under "outside
  authorized paths" by policy; this run used `.gitignore` files inside the
  authorized roots (`scripts/`, `data/`) instead.

## False-positive risks

- `read_likely_secret` on files whose names contain TOKEN/KEY/SECRET/credential.
- `protected_branch_push` matching `push origin main` inside a larger command
  (e.g. an `echo`).
- Force-push detection on `push +ref` style invocations is conservative.
See `scripts/README.md` for the full list.
