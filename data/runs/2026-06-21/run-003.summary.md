## Main Findings

1. Keep Markdown/YAML and GitHub as the canonical coordination layer for the
   current scale. Add databases as derived indexes or runtime state, not as a
   replacement for inspectable source-of-truth files.
2. Measure a single-agent baseline before adding multi-agent orchestration.
   Parallel workers should be introduced only for a demonstrated throughput
   bottleneck.
3. Build local structured telemetry and hard budget/termination controls before
   adopting an external observability platform or durable workflow engine.
4. Reuse distributed-systems primitives: leases, heartbeats, isolated
   worktrees, idempotency keys, bounded retries, and human approval gates.
5. Treat benchmark percentages, cost estimates, and upgrade thresholds as
   source-specific or provisional until self-evo has local measurements.
6. OpenViking is an active user deployment. This run did not resolve its public
   repository identity or benchmark evidence, so it is not classified as dead.
   The checked GBrain URL being unavailable is likewise not evidence that the
   project is abandoned.
