# First Scout Brief — AI Agent 内容方法论与首批选题

## 目标

基于可核验的公开材料，理解高质量 AI Agent 实践内容如何选择问题、组织证据、
形成标题和建立可信度，并为本项目提供 10 个可以实际完成的候选选题。

本次研究不是寻找一套保证增长的公式，也不以无法核验的粉丝量或阅读量作为结论。

## 核心问题

1. 哪些 AI Agent 内容提供了真实、可复用的实践价值？
2. 它们如何呈现问题、过程、证据、失败和结果？
3. 哪些标题和结构准确表达价值，而不是依赖夸张承诺？
4. 哪些公开反馈可以作为弱信号，哪些数据不能可靠比较？
5. self-evo 已有经历中，哪些主题具有独特证据和可讲述性？

## 研究范围

- 样本数量：20–30 个代表性账号、栏目、文章或系列；
- 优先语言：中文和英文均可；
- 优先主题：Agent 工程、工作流、评测、自治边界、真实项目复盘；
- 时间范围：优先最近 18 个月，同时允许纳入少量长期有效的经典材料；
- 来源必须公开、无需登录，并符合 `rules/RESOURCE_APPROVALS.yaml`。

## 来源优先级

1. 作者原文、项目文档和公开演讲；
2. 可核验的公开案例与代码仓库；
3. 有完整上下文的高质量二手分析；
4. 聚合榜单只用于发现线索，不能单独支撑结论。

不得：

- 猜测或编造阅读量、转化率、粉丝画像；
- 把单个爆款案例当成普遍规律；
- 使用需要登录、Cookie 或私人账号的内容；
- 发布、评论、私信或联系真实用户。

## 执行步骤

1. 写明搜索问题和纳入/排除标准；
2. 建立样本表，记录 URL、作者、日期、主题、证据形态和选择理由；
3. 按选题、标题、结构、证据、可信度和传播路径归纳模式；
4. 明确反例、失败模式和证据不足之处；
5. 使用方法论模板生成 v1；
6. 使用选题模板生成 10 个候选并排序；
7. 推荐前三名，但不得代替人类确定最终选题。

## 预算与停止条件

- 最多纳入 30 个样本；
- 最多进行 3 轮搜索扩展；
- 单轮新增有效模式少于 2 个时停止扩展；
- 任一来源需要登录、付费或额外凭证时停止访问并记录；
- 调研无法支持确定结论时，必须标记“证据不足”。

## 交付物

正式 run 应生成：

```text
projects/content-operator/exploration/
  source-log-<date>.md
  content-methodology-v1.md
projects/content-operator/backlog/
  topic-candidates-<date>.md
projects/content-operator/runs/
  <run-id>.summary.md
```

方法论和选题文件分别从项目模板复制。来源日志至少包含 URL、访问日期、保留理由
和支持的结论，避免留下无法复查的链接堆积。

## 人工决策点

调研完成后，Agent 只提交：

- 排名前三的选题；
- 每个选题的推荐理由、证据强度、工作量和风险；
- Agent 明确推荐的一项。

由人类选择第一个正式写作题目。未经选择，不进入文章初稿阶段。
